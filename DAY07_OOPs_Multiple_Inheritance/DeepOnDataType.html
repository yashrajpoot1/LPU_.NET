Data Types and Arrays in C#

In C#, every variable has a data type. Understanding types and arrays helps you write code that is safer, faster, and easier to maintain. This page covers beginner to advanced ideas like value types, reference types, arrays, boxing/unboxing, and safe type checks using is / as.

Note: C# runs on the CLR. Your code is compiled into IL and executed by a JIT compiler.

1) Data Types in C#

C# is a strongly typed language, meaning types are checked early and consistently. Data types decide how much memory is used and what operations are allowed. Choosing the right type makes programs faster and avoids bugs. C# types are broadly grouped into value types and reference types.

C# Types (Big Picture) --------------------- +----------------------+ | Types | +----------+-----------+ | +------------+------------+ | | Value Types Reference Types (store value) (store reference)
Example: Built-in data types

Common value types: int, long, float, double, decimal, bool, char, struct

Common reference types: string, object, array, class

using System;

class Demo
{
    static void Main()
    {
        int age = 25;
        double price = 99.99;
        bool isActive = true;
        char grade = 'A';

        string name = "Gopi";
        object anything = 10; // can store any type

        Console.WriteLine($"{name} | Age: {age} | Price: {price} | Active: {isActive} | Grade: {grade}");
    }
}
Try it Yourself »
CategoryExamplesTypical Use
Numeric int, long, float, double, decimal Counts, measurements, money (use decimal for money)
Text char, string Characters, names, messages
True/False bool Flags like isPaid, isValid
General object Generic storage (use carefully)
Beginner Tip: Use int for whole numbers and string for text. Use decimal for currency.
Advanced Note: Use the smallest appropriate numeric type for memory efficiency, but prioritize clarity. Also prefer explicit conversions when there is a risk of data loss.
Real-world use case

In a billing system: decimal stores money, DateTime stores invoice date, and bool tracks payment status.

2) Value Types and Reference Types

The biggest difference is how data is stored and copied. Stack is typically used for value-type storage (conceptually), while the heap stores objects for reference types. When you assign a value type, the value is copied. When you assign a reference type, the reference is copied, so two variables can point to the same object.

Copy Behavior ------------- Value Type: a = 10 b = a => b gets COPY of 10 (independent) Reference Type: a ---> [Object: "Hello"] b = a => b points to SAME object
Example: Copy behavior
using System;

class Demo
{
    static void Main()
    {
        // Value type copy
        int a = 10;
        int b = a;
        b = 99;
        Console.WriteLine($"a={a}, b={b}"); // a=10, b=99

        // Reference type copy (array is reference type)
        int[] x = { 1, 2, 3 };
        int[] y = x;
        y[0] = 999;
        Console.WriteLine($"x[0]={x[0]}, y[0]={y[0]}"); // both 999
    }
}
Try it Yourself »
Real-world use case

In a shopping cart app, a Cart object (reference type) is shared across pages. If one page adds an item, other pages see it too because they reference the same cart object.

Beginner Tip: If you see unexpected changes, check if you copied a reference (arrays/classes) instead of values.
Warning: Accidentally sharing references can create bugs (e.g., modifying the same array from multiple places). Use cloning when needed.
Advanced Note: Passing reference types to methods still passes the reference by value. Use ref only when you really need to reassign the caller’s variable.

3) Boxing and Unboxing

Boxing converts a value type into an object. Unboxing converts it back. Boxing/unboxing is common when using older APIs or non-generic collections. Modern C# uses generics to reduce boxing.

Boxing/Unboxing Flow -------------------- int (value) --(boxing)--> object (heap object) object --(unboxing)-> int (value)
Example: Boxing and Unboxing
using System;

class Demo
{
    static void Main()
    {
        int n = 42;

        object boxed = n;      // boxing
        int unboxed = (int)boxed; // unboxing (explicit cast)

        Console.WriteLine($"n={n}, boxed={boxed}, unboxed={unboxed}");
    }
}
Try it Yourself »
Warning: Unboxing the wrong type throws an exception. Example: (int)boxed fails if boxed actually contains a double.
Real-world use case

In logging frameworks, values may be stored as object for flexibility. Boxing can occur if you pass value types frequently—use structured logging and generics where possible.

Beginner Tip: Prefer List<int> over older collections like ArrayList to avoid boxing.
Advanced Note: Boxing creates allocations and can increase GC pressure. In high-performance systems, avoid boxing in hot loops.

4) Arrays in C# (1D, 2D, Jagged)

An array stores multiple values of the same type under one variable name. Arrays are fixed-size, so you decide the length when creating them. Use arrays when the size is known and fast indexing is required. For dynamic sizing, prefer List<T> (covered in Collections topics).

Array Index View (1D) --------------------- scores: [ 85 ][ 90 ][ 72 ][ 88 ] index : 0 1 2 3
Example: Single Dimensional Array (1D)
using System;

class Demo
{
    static void Main()
    {
        int[] scores = { 85, 90, 72, 88 };

        Console.WriteLine("All Scores:");
        for (int i = 0; i < scores.Length; i++)
            Console.WriteLine($"scores[{i}] = {scores[i]}");

        // Average
        int sum = 0;
        foreach (var s in scores) sum += s;
        Console.WriteLine("Average = " + (sum / (double)scores.Length));
    }
}
Try it Yourself »
Example: Multi-Dimensional Array (2D)
2D Array (Matrix) ----------------- marks[rows, cols] [ 78 81 90 ] [ 65 70 72 ] 
using System;

class Demo
{
    static void Main()
    {
        int[,] marks = {
            { 78, 81, 90 },
            { 65, 70, 72 }
        };

        Console.WriteLine("Matrix:");
        for (int r = 0; r < marks.GetLength(0); r++)
        {
            for (int c = 0; c < marks.GetLength(1); c++)
                Console.Write(marks[r, c] + " ");
            Console.WriteLine();
        }
    }
}
Try it Yourself »
Example: Jagged Array (array of arrays)

A jagged array is flexible because each row can have different size.

Jagged Array ------------ days[0] -> [31 items] days[1] -> [28 items] days[2] -> [31 items] 
using System;

class Demo
{
    static void Main()
    {
        int[][] data = new int[3][];
        data[0] = new int[] { 1, 2, 3 };
        data[1] = new int[] { 10, 20 };
        data[2] = new int[] { 7, 8, 9, 10 };

        for (int i = 0; i < data.Length; i++)
        {
            Console.Write("Row " + i + ": ");
            foreach (var v in data[i]) Console.Write(v + " ");
            Console.WriteLine();
        }
    }
}
Try it Yourself »
Real-world use case
  • 1D: Daily sales for 30 days
  • 2D: Seating chart (rows × columns)
  • Jagged: Monthly calendar (different days per month)
Beginner Tip: Use 1D arrays first. Move to 2D/Jagged only when the problem truly needs rows/columns.
Advanced Note: 2D arrays are rectangular and memory-contiguous-ish; jagged arrays are arrays of references. Choose based on access patterns and memory layout needs.

5) Nullable Types

Some value types (like int, double, bool) cannot hold null. A nullable type solves this by allowing missing/unknown values. This is extremely common in database-driven applications where columns can be empty. Use `T?` for nullable value types and handle them safely using `??` and `HasValue`.

Nullable Concept ---------------- int age = 25 -> always has a number int? age = null -> can be missing (unknown)
Example: Nullable int + Null-Coalescing
using System;

class Demo
{
    static void Main()
    {
        int? discount = null;  // might come from DB

        // If discount is null, use 0
        int finalDiscount = discount ?? 0;

        Console.WriteLine("Final Discount = " + finalDiscount);

        if (discount.HasValue)
            Console.WriteLine("Discount is " + discount.Value);
        else
            Console.WriteLine("No discount available");
    }
}
Try it Yourself »
Real-world use case

In a user profile system, MiddleName can be missing (null), and Age might be unknown. Nullable types let you represent missing values without fake defaults.

Beginner Tip: Use `??` to set a default when data is missing: value ?? defaultValue.
Advanced Note: In modern C#, nullable reference types (NRT) help you reduce NullReferenceException. But `T?` for value types is still the classic nullable mechanism.

6) Implicitly Typed Local Variables (var)

`var` allows the compiler to infer the variable’s type from the assigned value. This is called type inference. `var` is useful for readability when the type is obvious or very long (like LINQ queries). However, the type is still determined at compile-time, not at runtime.

var Rule (Simple) ----------------- var x = 10; // x becomes int (fixed) x = "hi"; // ❌ not allowed (still int)
Example: var in action
using System;

class Demo
{
    static void Main()
    {
        var count = 10;          // inferred as int
        var price = 99.5;        // inferred as double
        var name = "ItTechGenie"; // inferred as string

        Console.WriteLine($"{count} | {price} | {name}");
    }
}
Try it Yourself »
Real-world use case

In LINQ queries, `var` keeps code readable: the result type might be an anonymous type, which you cannot name directly.

Beginner Tip: Use `var` only when the right-hand side clearly shows the type.
Advanced Note: `var` can improve maintainability—if you change the right side type, the left updates automatically. But overusing it can reduce clarity.

7) var vs dynamic

`var` is resolved at compile-time (fixed type). `dynamic` is resolved at runtime (type checking happens during execution). `dynamic` can be useful for COM interop, reflection-heavy code, or dynamic JSON objects, but it can also introduce runtime errors if misused.

var vs dynamic -------------- var -> compile-time type (safe) dynamic -> runtime type (flexible, but risky)
Example: var vs dynamic
using System;

class Demo
{
    static void Main()
    {
        var a = 10;        // int
        // a = "hi";       // ❌ compile-time error

        dynamic b = 10;    // can change at runtime
        b = "hi";          // ✅ allowed
        Console.WriteLine(b);

        // Risk: runtime error if method doesn't exist
        // Console.WriteLine(b.NotARealMethod()); // would fail at runtime
    }
}
Try it Yourself »
Real-world use case

When consuming a dynamic JSON payload (unknown schema), `dynamic` may reduce boilerplate. In enterprise code, prefer strongly typed models for reliability.

Warning: `dynamic` hides compile-time safety and can cause runtime crashes. Use it only when you truly need dynamic behavior.
Advanced Note: Many `dynamic` operations use the Dynamic Language Runtime (DLR) under the hood, which adds overhead. In performance-critical code, avoid it.

8) is and as Operator

In OOP, you often work with base types and derived types. The is operator checks if an object can be treated as a particular type. The as operator attempts a cast and returns null if it fails (for reference types / nullable value types). This helps you avoid invalid cast exceptions.

Safe Type Handling ------------------ obj -> check with 'is' obj -> try cast with 'as' -> null check -> use
Example: is and as
using System;

class Animal { public virtual void Speak(){ Console.WriteLine("..."); } }
class Dog : Animal { public override void Speak(){ Console.WriteLine("Woof"); } }

class Demo
{
    static void Main()
    {
        Animal a = new Dog();

        // is check
        if (a is Dog)
            Console.WriteLine("a is a Dog");

        // as cast
        Dog d = a as Dog;
        if (d != null)
            d.Speak(); // Woof
    }
}
Try it Yourself »
Real-world use case

In a payment system, you might have Payment base class with derived types like CardPayment and UPIPayment. Use is/as to safely handle type-specific fields without runtime crashes.

Beginner Tip: Prefer `is` for checking and branching; use `as` only when you want null instead of exception.
Advanced Note: In modern C#, pattern matching (`if (a is Dog dog)`) is cleaner and safer than separate `is` + cast.

Best Practices
  • Prefer decimal for currency and financial calculations.
  • Use nullable types when data can be missing (DB fields, optional input).
  • Avoid boxing in performance-critical code; prefer generics (e.g., List<int>).
  • Use arrays when size is fixed; use List<T> when size changes.
  • Use `var` for readability, not to hide type meaning.
  • Use `dynamic` carefully—keep it isolated and validated.
  • Prefer safe checks using `is` and pattern matching to avoid invalid cast exceptions.

Step-by-step Lab (Hands-on)

Build a small console flow that stores weekly hours, handles missing data, and prints a summary. This lab connects multiple subtopics together (types + arrays + nullable + var).

Steps
  1. Create a `double[]` with 4 weeks of hours.
  2. Allow one week to be missing using `double?`.
  3. Compute total and average safely using `?? 0`.
  4. Print a report.
using System;

class Lab
{
    static void Main()
    {
        double[] hours = { 40, 42, 38, 45 };
        double? week2BonusHours = null; // missing data example

        double total = 0;
        foreach (var h in hours) total += h;

        total += (week2BonusHours ?? 0);

        double avg = total / hours.Length;

        Console.WriteLine("Total Hours = " + total);
        Console.WriteLine("Average Hours/Week = " + avg);
    }
}
Beginner Tip: Print values after each step to understand what your program is doing.
Advanced Note: In real apps, this data often comes from DB/API; null handling prevents crashes.

FAQs
Is string a value type or reference type?

string is a reference type, but it behaves like immutable value in many scenarios because it is immutable.

Should I use arrays or List<T>?

Use arrays for fixed size and fast indexing. Use List<T> when size grows/shrinks.

Does var mean dynamic?

No. var is compile-time inferred. dynamic is runtime resolved.

When does boxing happen?

When a value type is converted to object/interface type, especially with non-generic APIs.

Can nullable type be used with all value types?

Yes. You can use T? for most value types (int?, double?, bool?, DateTime?).

Interview Questions (with short answers)
  1. What is the difference between value type and reference type?
    Value types store actual value; reference types store reference to an object.
  2. What is boxing and why is it expensive?
    It allocates a new object on heap and adds GC overhead.
  3. What is a jagged array?
    An array of arrays where each row can have different length.
  4. When to use decimal?
    For money/currency calculations to reduce rounding errors.
  5. What is nullable type?
    Allows value types to represent “no value” using T?.
  6. var vs dynamic?
    var is compile-time typed; dynamic is runtime typed.
  7. How do you safely cast in C#?
    Use `is` / pattern matching or `as` with null check.
  8. What is the default value of int and int??
    int default is 0; int? default is null.
  9. How to get length of 2D array?
    Use GetLength(dimensionIndex).
  10. Why avoid object for numeric data?
    It can cause boxing/unboxing and lose compile-time safety.

To Become Best coder Practice this

Practice makes you strong in interviews. Below are 15 coding questions (Easy → Hard) focused on data types, arrays, nullable, var/dynamic, boxing/unboxing, and is/as.

Easy (5)

E1) Count positives and negatives

Problem: Given N integers, count how many are positive and how many are negative.

Input: N, then N integers
Output: positivesCount negativesCount

Example Input: 5 → 2 -1 0 7 -3
Example Output: 2 2

Constraints: 1 ≤ N ≤ 10^5

Hint: Loop through array; update two counters.

E2) Find maximum in an array

Problem: Read N numbers and print the maximum.

Input: N, then N integers
Output: maxValue

Example Input: 4 → 10 90 45 12
Example Output: 90

Constraints: N ≥ 1

Hint: Initialize max = first element.

E3) Convert Celsius to Fahrenheit

Problem: Read a double Celsius and print Fahrenheit (F = C*9/5 + 32).

Input: C (double)
Output: F (double)

Example Input: 0
Example Output: 32

Constraints: -1000 ≤ C ≤ 1000

Hint: Use double for precision.

E4) Nullable discount

Problem: Read a price and an optional discount (may be blank). If discount missing, treat as 0.

Input: price, discountOrBlank
Output: finalPrice

Example Input: 100, blank
Example Output: 100

Constraints: price ≥ 0

Hint: Use nullable double? and null-coalescing.

E5) Reverse an array

Problem: Reverse N integers and print them.

Input: N then N ints
Output: reversed array

Example Input: 3 → 5 6 7
Example Output: 7 6 5

Constraints: N up to 10^5

Hint: Swap i and n-1-i.

Medium (5)

M1) Weekly sales average (1D)

Problem: Store 7 days sales and print total + average (double).

Input: 7 integers
Output: total average

Example Input: 10 20 30 40 50 60 70
Example Output: 280 40

Constraints: sales ≥ 0

Hint: total/7.0 for average.

M2) Student marks table (2D)

Problem: Read R students and C subjects, print each student total.

Input: R, C, then R×C marks
Output: totals per row

Example Input: R=2 C=3 → 10 20 30 5 5 5
Example Output: 60 15

Constraints: 1≤R,C≤50

Hint: Nested loops, sum each row.

M3) Jagged monthly attendance

Problem: Month has weeks with different working days. Store attendance per week (jagged array) and print total days present.

Input: W, then each week length and values
Output: totalPresent

Example Input: W=2; week1:3(1 1 0), week2:2(1 1)
Example Output: 4

Constraints: values are 0/1

Hint: Sum all inner arrays.

M4) Avoid boxing in mixed inputs

Problem: You receive numbers as strings. Convert safely to int and store in int[] (no object array).

Input: N then N strings
Output: validSum (ignore invalid)

Example Input: 4 → 10 abc 20 -5
Example Output: 25

Constraints: N up to 10^5

Hint: Use int.TryParse.

M5) Type filter using is/as

Problem: Given a list of objects representing messages, print only string messages.

Input: N then N values (some strings, some numbers)
Output: only strings

Example Input: 4 → "hi" 10 "ok" 2.5
Example Output: hi ok

Constraints: N ≤ 10^4

Hint: Use `is string` or `as string` with null-check.

Hard (5)

H1) Rotate array K times

Problem: Rotate an array right by K positions and print final array.

Input: N, K, then N ints
Output: rotated array

Example Input: N=5 K=2 → 1 2 3 4 5
Example Output: 4 5 1 2 3

Constraints: N up to 10^5

Hint: K %= N; use extra array or reverse method.

H2) Missing values report (nullable + arrays)

Problem: Read N sensor readings where -1 means missing. Convert to int?[] and print count of missing + average of valid.

Input: N then N ints
Output: missingCount validAverage

Example Input: 5 → 10 -1 20 -1 30
Example Output: 2 20

Constraints: N up to 10^5

Hint: Convert -1 to null, sum only non-null.

H3) Matrix diagonal analysis (2D)

Problem: For an NxN matrix, print main diagonal sum, secondary diagonal sum, and their difference.

Input: N then N×N ints
Output: d1 d2 absDiff

Example Input: N=3 → 1 2 3 4 5 6 7 8 9
Example Output: 15 15 0

Constraints: N ≤ 500

Hint: d1 += a[i,i], d2 += a[i,n-1-i].

H4) Jagged salary bands

Problem: Departments have different employee counts. Store salaries in jagged array and print department-wise max salary.

Input: D, then each dept size + salaries
Output: max per dept

Example Input: D=2; dept1:3(10 20 15), dept2:2(50 40)
Example Output: 20 50

Constraints: D ≤ 50, each dept ≤ 10^4

Hint: For each inner array, scan for max.

H5) Detect boxing hotspots

Problem: You have a legacy object[] storing numbers. Convert it to int[] safely and report how many invalid items existed.

Input: N then N items (some not ints)
Output: convertedSum invalidCount

Example Input: 5 → 10 "x" 20 30 2.5
Example Output: 60 2

Constraints: N ≤ 10^5

Hint: Use `is int` to filter; unbox only when compatible.
